# Build the Next AI Trader 交易未来

## 赛道背景

AI 已经学会了回答问题。它会聊天、会写代码、会生成图片。

但现实世界真正有价值的工作，从来不是回答一个问题。

而是理解需求、寻找信息、调用工具、不断验证、与其他人协作，最终完成一项任务。

我们相信，未来的软件将不再只是 App，而是一群能够自主协作的 Agent。

而金融，是检验 Agent 能力最好的实验场。

它拥有海量实时数据、复杂决策链路、严格反馈机制，最重要的是——这里没有唯一正确的标准答案。

如果一个 Agent 能够在这里完成工作，它未来也许也能够进入科研、医疗、制造、法律等更多领域。

所以，这不仅是一个金融科技领域的赛道，更是一场关于未来 Agent 的实验。

## 我们期待什么？（赛道要求）

本赛道希望探索 AI Agent 如何在真实金融场景中完成复杂决策任务。

交易从来不是一个动作，而是一系列复杂决策的协作过程。

理解需求、获取数据、分析信息、构建策略、评估风险、验证结果……未来，这些工作或许将不再由人独立完成，而是一支由多个 AI Agent 组成的团队共同协作。

我们鼓励你重新定义 AI 在投研、策略开发与交易中的角色。你可以构建单个 Agent，也可以设计完整的 Multi-Agent 系统；可以专注于某一个环节，也可以重构整个工作流。

## 你可以尝试的方向

以下方向仅供参考，并非限制。

### ① Alpha Discovery Agent｜因子研究智能体

让 AI 像研究员一样发现市场规律。

你的 Agent 可以自主完成因子构建、数据处理、因子评价、IC / Rank IC 分析、分组回测、因子解释等研究流程，帮助研究者从“提出假设”走向“验证 Alpha”。

### ② Strategy Agent｜策略开发智能体

让一句自然语言，变成一个完整的策略。

从需求理解、策略生成，到自动调用回测能力、分析收益表现、风险暴露、换手率等指标，并持续优化策略，让 AI 成为真正的策略开发伙伴。

### ③ Research Agent｜投研智能体

让 AI 拥有真正的研究能力。

围绕行业、公司、宏观经济、市场风格等主题，自主完成数据检索、信息整合、归因分析、图表生成、研究摘要甚至投资观点输出，打造下一代 AI Research Assistant。

### ④ Portfolio Agent｜组合管理智能体

让 AI 学会管理，而不仅仅是分析。

围绕资产配置、风险控制、持仓优化、组合再平衡、绩效归因等真实场景，构建能够持续辅助投资决策的 Portfolio Manager。

### ⑤ Multi-Agent Workflow｜多智能体协作系统

真正复杂的问题，从来不是一个 Agent 能独立完成的。

你可以构建一支 AI 投研团队，例如：

- Planner Agent（任务规划）
- Data Agent（数据获取）
- Research Agent（信息分析）
- Factor Agent（因子研究）
- Strategy Agent（策略生成）
- Backtest Agent（策略验证）
- Report Agent（研究报告）

探索 Agent 之间如何协作、通信、分工、反馈与持续优化，重新定义未来的 AI 工作流。

### ⑥ Open Innovation｜开放创新方向

未来交易员，不一定长成今天的样子。

任何能够体现 Agent 自主决策、多智能体协作、工具调用（Tool Use）、数据融合、Workflow 编排、金融 AI 创新应用的作品，我们都欢迎。

如果你的想法足够大胆，请把它做出来。

## 我们更希望看到

相比功能的多少，我们更关注：

- Agent 是否真正能够完成一项复杂任务，而不仅仅是聊天。
- 多个 Agent 是否形成了高效的协作机制，而不仅仅是简单串联。
- 是否充分利用数据、工具与工作流，展现 AI 在真实金融场景中的价值。
- 是否提出了一个我们从未见过、却相信未来可能发生的产品。

## 评选标准

参赛作品须以 A2A Remote Agent 形式提交。参赛者需自行托管智能体服务，并向平台提交可访问的 Agent Card URL。

提交内容包括：

- Agent 名称、简介、团队信息
- Agent Card URL，例如：https://example.com/.well-known/agent-card.json
- 智能体服务地址及鉴权方式
- 作品说明文档
- 示例问题与预期输出
- 使用的数据 Skills / 投研 Skills 列表
- 如有开源代码，可提交代码仓库地址

智能体需满足以下基本要求：

- 支持 A2A 协议调用。
- Agent Card 信息完整、真实、可访问。
- 服务在评审期间保持稳定在线。
- 能处理平台发送的自然语言任务。
- 输出过程和最终结果应清晰、可解释。
- 不得绕过平台权限直接访问未授权数据。
- 底座模型统一使用 DeepSeek V4 Pro 进行开发。
- Agent 总响应时间（从收到任务到完成结果返回）不应超过 20 分钟。

## A2A 接入规范

参赛智能体应提供标准 Agent Card，用于描述智能体身份、能力、技能、输入输出模式、鉴权方式和服务端点。

建议至少包含：

```json
{
  "name": "Factor Research Agent",
  "description": "An agent for factor analysis and backtesting",
  "version": "1.0.0",
  "skills": [
    {
      "name": "factor_analysis",
      "description": "Analyze factor performance using provided research skills"
    },
    {
      "name": "strategy_backtest",
      "description": "Run strategy backtests and summarize results"
    }
  ],
  "capabilities": {
    "streaming": true,
    "pushNotifications": false
  }
}
```

平台将根据 Agent Card 自动识别智能体能力，并在评审任务中调用对应 Agent。
